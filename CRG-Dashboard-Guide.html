1. What the app is

The CRG Dashboard (internally also called the "CRG Platform") is an internal operations hub built for Cobb Realty Group. It gives the whole team one home base to:

  • Manage real-estate transactions / deals through their full pipeline
  • Browse a team directory with contact details
  • Access training resources and SOPs
  • Run new-agent onboarding with task tracking and approvals
  • Post company announcements
  • Reach important forms

It is a mobile-first app: it's designed to look and feel like a phone app. On a computer it appears centred in a narrow, phone-shaped frame. It is a PWA (Progressive Web App), meaning each person can install it onto their phone's home screen and open it like a normal app — but it's delivered over the web, with no app store involved.

2. The technology behind it (in simple terms)

  • The app itself is hosted on Vercel — a service that publishes the website online over a secure (HTTPS) connection.
  • All the data, the user logins, and uploaded files are handled by Supabase — a cloud service that provides three things: a database (where information is stored), an authentication system (logins/passwords), and file storage (for photos and documents).
  • Everything you see in the app is stored in the Supabase database. Uploaded files live in two storage areas: one for team member photos, one for onboarding documents.

3. Accounts: logging in, registering, passwords & email

Logging in

People sign in with their email and password. Anyone who isn't logged in is automatically sent to the login screen when they try to open an internal page. Anyone already logged in who opens the login or signup screen is sent straight to the dashboard.

Registering (signing up) — and the key restriction

To register, a person enters their full name, email, password (at least 8 characters) and a confirmation.

Important — the email gate: A person can only register if their email has already been added to the Team Directory as an active member. If it hasn't, signup is blocked with the message: "This email is not registered with the team. Please contact your admin to be added first."

So the correct order is always: (1) you (Super Admin) add the person to the Team Directory with their email → (2) that person can then create their account with that same email.

When someone signs up successfully, the system automatically creates their profile and assigns them the "New Agent" role, with onboarding status set to "not started."

Forgot password

A user clicks "Forgot password?" on the login screen and enters their email. The system emails them a secure reset link. Clicking the link opens a "Set a new password" screen where they choose a new password (at least 8 characters) and confirm it — then they're signed in. This works fully, end to end.

Two notes: (1) For security, the reset link must be opened in the same web browser the request was made from, and it expires after a while — if it's expired, the screen offers a "Request a new link" button. (2) As Super Admin you can also reset anyone's password directly from the Supabase Dashboard (Authentication → Users → select the user), which is handy if someone is fully locked out.

Changing password (while logged in)

On the Profile page, a user enters their current password (which is re-checked), then a new password (at least 6 characters) and a confirmation.

Changing email (needs your approval)

On the Profile page a user can request an email change. This does not happen immediately — it creates a request that an admin must approve. (Full flow in the Email Change Requests screen below.)

What each switch unlocks:

  • View ALL transactions — see every deal, not just your own.
  • Edit transactions — add, edit, delete deals and change a deal's stage.
  • Manage users — unlocks the Email Change Requests page. (Only Super Admin & Operations Manager.)
  • Manage onboarding — see and approve/reject new agents in onboarding. (Only Super Admin & Operations Manager.)

  • Assigning a role to a team member
  • Opening the Manage Roles page
  • Adding / editing / deleting training resources
⚠️ Good to know: the permission switches are editable, so the table above reflects today's settings — you can change most of them anytime on the Manage Roles page. Note that with the current settings, every role can already view all transactions, so the "see only my own deals" behaviour isn't active right now.

6. Every screen, explained

How you move around: the main menu is a bar at the bottom of the screen with Home, Deals, Team, Training (only if your role can access training) and Forms. Other screens — Profile, Onboarding, and Manage Roles — are reached from the menu that opens when you tap the round avatar in the top-right. A bell icon beside it opens notifications.

(a) Home (Dashboard)

  • A welcome banner and three summary numbers: Active, Pending, Closed deals (calculated from transactions).
  • Reports If the role can View Reports: reporting sections appear — Pipeline by Stage (count + dollar value per stage), Agent Performance (ranked by deals and total volume), and Deals Closed by Month (a 6-month bar chart).
  • For a New Agent not yet approved: a reminder card to complete onboarding.
  • A team video (embedded YouTube).
  • Team Updates — company announcements. Announcements If the role can Manage Announcements, they can add, edit and delete these here.
  • Quick Access tiles to Deals, Team, Training and Forms.
  • An embedded company Google Calendar.
  • Social media links (Facebook, Instagram, LinkedIn).

Reports appear here on the Home screen — there is no separate Reports page.

(b) Transactions ("Deals")

  • The deal pipeline: a row of counts per stage, filter buttons (All, Active, Pending, Closed, Lost) and a search box (by client, address or agent).
  • Who sees what: roles with View ALL transactions see every deal; otherwise a person would see only deals assigned to them. (Currently all roles view all.)
  • Edit If the role can Edit Transactions: an Add button, plus edit, delete, and a quick stage-change dropdown on each deal.
  • Each deal stores: property address, client name, client email, client phone, the assigned agent (chosen from the Team Directory), stage, price, listing/offer/contract/closing dates, notes, a Zillow link, and a Google Drive folder link.
  • The 10 stages: Lead, Active, Offer, Pending, Under Contract, Closing, Closed, Archived, Lost, Withdrawn.
  • Tapping a deal opens a detail panel. If no Zillow link is provided, the app builds one from the address.

(c) Team Directory (titled "Agent Directory")

  • A searchable list of active team members as cards (photo or initials, name, email, and quick buttons to open their Drive folder, call, or email them).
  • Super Admin only Adding a member. Fields: full name, email, phone, photo (upload, max 5MB — auto-compressed), Google Drive folder link, and — only when editing someone who already has a login — a Role selector (changeable by Super Admin only).
  • Editing: the Super Admin can edit anyone; a regular user can edit only their own card.
  • Super Admin only Deleting a member.
This directory is the gatekeeper for registration. Adding a person here (with their email) is exactly what allows that person to sign up for an account. This is the most common first task for you as Super Admin.

(d) Training & Support

  • Open to any role that can Access Training (all roles currently can).
  • Lists training resources grouped by category, with category filter buttons. Each resource shows a title, description, type (video / document / link / course), an optional "Required" badge, and a link to open it.
  • Super Admin only Adding / editing / deleting resources.

(e) Forms & Documents

  • A list of important links (Google Forms, documents, etc.) that open in a new tab. Each entry has a title, URL, and an optional description.
  • Available to everyone logged in; no special permission needed to view.
  • Manage forms If the role can Manage Forms: an Add button appears in the header, plus edit and delete buttons on each form. Forms are stored in the database and ordered automatically.
  • Two forms are pre-loaded: "Accepted Offer & Closing Form" and "Transaction Update Form."

(f) Onboarding

  • For a New Agent: a progress bar and task sections — Getting Started, Licensing & Compliance, Training, Final Steps. Tasks can be check-off items, document uploads (max 10MB), videos, or forms. When all required tasks are done, a "Submit for approval" button appears; submitting marks them "pending approval" and notifies the admins.
  • Manage onboarding For admins (Super Admin & Operations Manager): an "Onboarding Management" view listing New Agents and their progress; each expands to show their tasks and uploaded documents. For anyone pending approval, the admin can Approve (which promotes them to the "Agent" role and notifies them) or Reject.
  • For everyone else (already-approved staff): a read-only "Onboarding Complete" history.
  • The 11 standard onboarding tasks: (1) Welcome to CRG video, (2) Complete Profile, (3) Review Company Policies, (4) Upload Real Estate License, (5) E&O Insurance proof, (6) W-9 Form, (7) CRM Training, (8) Transaction Process Overview, (9) Marketing Guidelines (optional), (10) Schedule Orientation Call, (11) Submit Onboarding Completion Form.
⚠️ Good to know: rejecting an agent updates their status but does not currently send them a notification (they will, however, see a "not approved" message on their own onboarding screen).

(g) Profile

  • Shows avatar (initials), name, role, and member-since date.
  • The user can edit their Full Name and Phone directly (saved immediately).
  • Changing Email submits an approval request (not immediate) and shows its status (pending / approved / declined).
  • Change Password: current password (re-verified) + new password (min 6 characters) + confirmation.

(h) Email Change Requests Manage users

  • Open to roles that can Manage Users (Super Admin & Operations Manager); others see a "no permission" message.
  • Shows pending email-change requests with Approve / Decline buttons, plus a history of past decisions.
  • When approved, the system updates the person's actual login email, their profile, and their directory entry, then notifies them. When declined, it notifies them it was declined.

7. Manage Roles — your control panel for access Super Admin only

This is the screen where you decide what each role is allowed to do. Reach it from the top-right avatar menu → Manage Roles. Anyone who isn't a Super Admin who tries to open it sees "You don't have permission to view this page."

How it works:

  • It lists every role except Super Admin (your own role always keeps full access and can't be limited).
  • Tap a role to expand it, then flip switches on or off. Changes take effect immediately for everyone with that role.
  • The six switches you can toggle here are: View All Transactions, Edit Transactions, View Reports, Manage Announcements, Access Training, and Manage Forms.
⚠️ Note: three settings are not adjustable from this screen — Manage Users, Manage Onboarding, and the Must-onboard flag. Those are fixed by the role itself. Likewise, the Super-Admin-only actions (adding/deleting team members, assigning roles, managing training) cannot be granted to other roles from here — they are tied to the Super Admin role.

So, in practice, the access you can hand out per section from Manage Roles is: who can see reports, who can edit deals, who can post announcements, who can open training, and whether a role sees all deals. For who can manage onboarding or approve email changes, you would change the person's role (e.g. make them an Operations Manager) on the Team Directory.

8. Notifications

  • A bell icon in the top bar shows notifications in real time, with a red unread-count badge. Users can mark all as read, or tap one to jump to the related page.
  • Notifications are created automatically when: someone requests an email change (admins notified), an email change is approved or declined (user notified), a New Agent submits onboarding for approval (admins notified), and a New Agent is approved (user notified).

9. The data behind the app

In simple terms, the information is organised into these areas of the database: user profiles, roles, the team directory, transactions/deals, announcements, onboarding tasks, each person's onboarding progress, training resources, forms & documents, email-change requests, and notifications. Logins are kept separately in the secure authentication system. Uploaded files live in two storage areas: team photos and onboarding documents.

10. Important things to be aware of

Security model — please read. The app correctly hides buttons and pages based on each person's role, so in normal day-to-day use everyone is limited to what they should do. However, the deeper database protections are currently permissive — they mainly check that a person is logged in, rather than enforcing each role's limits at the data level. In plain terms: the safeguards live in the app's screens rather than deep in the database.

This is generally fine for a small, trusted internal team. But it means a technically-skilled logged-in user could, in theory, reach data the screens would normally hide. If stricter separation is ever needed, ask whoever maintains the app to tighten the database security rules. (One part is already properly locked down at the data level: the email-change approval process.)

Uploaded files are publicly readable by link. Onboarding documents (licenses, insurance, W-9) and team photos are stored in areas readable by anyone who has the direct file link. Treat them as not strictly private, and keep that in mind before uploading sensitive documents.
  • The self-service password reset works end to end (see Section 3). Reminder: the link must be opened in the same browser it was requested from, and it expires after a while.
  • There is no separate "Reports" or "Settings" page — reports appear on the Home screen.

11. Quick-start checklist for the Super Admin

Your most common tasks, at a glance:

  1. Onboard a new person: Team Directory → Add → enter their name & email (and details). This is what lets them register. Tell them to sign up with that exact email.
  2. Approve a new agent: Onboarding → find the agent (pending approval) → Approve (this promotes them from "New Agent" to "Agent"). Or Reject.
  3. Approve / decline an email change: open Email Change Requests (avatar menu or the notification) → Approve or Decline.
  4. Adjust what a role can do: Manage Roles → expand a role → flip the switches (reports, edit deals, announcements, training, view-all deals, manage forms).
  5. Change someone's role entirely: Team Directory → edit that member → set their Role (e.g. promote to Operations Manager). Needed for granting "manage users / manage onboarding."
  6. Post an announcement: Home → Team Updates → Add.
  7. Manage training resources: Training → Add / Edit / Delete.
  8. Manage forms & documents: Forms → Add / Edit / Delete. (Any role with the "Manage Forms" switch can do this too.)
